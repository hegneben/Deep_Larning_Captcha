# Contributing

Thank you for contributing to this project.

## Workflow

- Create a new branch from `develop`
- Use feature branches (`feature/...`) for new changes
- Keep changes focused on a single task
- Write clear and maintainable code
- Test your changes before pushing

## Pull Requests

- Open a pull request to `develop`
- Provide a clear title and description
- Link the PR to an issue if possible
- Ensure the code runs correctly
- Make sure the code is understandable and documented when needed

## Review Process

- At least one team member must review the PR before merging
- Address review feedback before approval
- Only merge when all checks pass

## Code Style

- Follow consistent Python formatting
- Use meaningful variable and function names
- Keep functions small and readable
- Avoid unnecessary complexity

## Commit Messages

Use descriptive commit messages.
